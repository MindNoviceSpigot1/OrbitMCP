# Contributing to OrbitMCP

Thanks for your interest in improving OrbitMCP! Contributions of all sizes are welcome — bug fixes, new provider integrations, documentation, and feature ideas.

## Before you start

- Check open [Issues](../../issues) and [Pull Requests](../../pulls) to avoid duplicate work.
- For larger features (new chains, new providers, trading actions), open an issue first to discuss the approach before writing code.

## Setting up locally

1. Fork the repo and clone your fork.
2. Install dependencies (see README for platform-specific setup).
3. Create a branch: `git checkout -b feature/short-description`.

## Making changes

- Keep pull requests focused on a single change — smaller PRs are reviewed faster.
- Follow the existing code style in the file you're editing.
- Add or update documentation if your change affects usage (README, examples, config).
- If you're adding a new provider or chain, include a short note in the PR description about rate limits, API key requirements, and data reliability.

## Submitting a pull request

1. Push your branch and open a PR against `main`.
2. Fill out the PR template — describe what changed and why.
3. Link any related issues.
4. A maintainer will review and may request changes before merging.

## Reporting bugs

Please use the bug report issue template and include:
- OS and OrbitMCP version
- The AI client you're using (Claude, ChatGPT, Cursor, etc.)
- Steps to reproduce
- Expected vs actual behavior

## Code of Conduct

By participating, you agree to follow our [Code of Conduct](CODE_OF_CONDUCT.md).
